## Working agreements

Targets LilyGo T4 AMOLED display
Uses the graphics library TFT_eSPI
All libraries to use are specified in platformio.ini
There is a reference application developed for the M5 Stick C Plus in the directory parallel to this project called m5-i2s. Note that the I2S microphone is a different type to the one used in this project. It includes fft files which are duplicated in this project. It also
contains FFT display code and I2S read code (but for a PDM mic).

## Project Description

t4-i2s is an application which:

- Targets LilyGo T4 AMOLED ESP32-S3 development board
- Reads I2S sound data from an Adafruit I2S MEMS Microphone Breakout - SPH0645LM4H
- Stores microphone data to SD card without allowing SD write latency to block I2S recording or FFT display updates
- Has an FFT display to show recording activity and playback activity
- Has an Oscilloscope (time domain) display to show recording activity and playback activity.
- Has a spectrum analyzer display  (like on a hifi music system) to show recording activity and playback activity.
- Uses the m5-i2s project as an initial reference for I2S based recording and FFT display.
- Has WiFi access, but WiFi, web server, and FTP features are not part of the current implementation priority.
- Will later use ESPAsyncWebServer to implement a web server for controlling record and playback.
- Will later use SimpleFTPServer to allow users to download sound files.

## Specifications

- The AMOLED display has dimensions 600 x 450 pixels, landscape.
- The FFT graphic is to be 200 x 135 pixels, landscape and centralised.
- The Oscilloscope graphic is to be 200 x 135 pixels, landscape and located immediately below the FFT graphic.
- The spectrum analyzer is to be 200 x 135 pixels, landscape and located immediately above the FFT graphic. The number of bands and the lower and upper frequencies to display, the width of each band and the smoothing is to be definable.
- The SD card is formatted FAT32 and has size 16GB.
- Recordings are mono WAV files using 16 kHz sample rate and 16-bit PCM samples.
- Each new recording is to have a new file. File naming can be sequential, eg 1.wav
- FreeRTOS tasks are used for concurrent I2S recording and FFT display.
- SD card recording must be buffered/asynchronous so SD write latency does not block I2S capture or FFT display work.
- Playback does not output to audio hardware yet; it reads the stored WAV file and shows playback activity on the FFT display to verify stored data is readable.
- The WiFi credentials are stored in a mercator_secrets.c file which you must not read.
- The template for the secrets file is called mercator_secrets_template.c. Variables are commented out so they don't clash with the real values in mercator_secrets.c. Assume variables in the real secrets file have same name and type as in the template file.
- The user must be able to select record or playback and if playback pick a file from a list shown on the screen.
- There are two button GPIOs, with buttons that are normally open and connect to ground on button press.
    GPIO 48: Top Button GPIO
    GPIO 21: Side Button GPIO
- Top button is the primary control and side button is the secondary control. Choose a satisfactory usage pattern for record/playback navigation.
- The I2S GPIOs are:
    GPIO 39: BCLK           (relative to ESP32-S3 this is output: bit clock generated by ESP32-S3)
    GPIO 40: LRCLK          (relative to ESP32-S3 this is output: Left/Right clock / word select)
    GPIO 41: Data (DOUT)    (relative to ESP32-S3 this is Input, Mic Data --> ESP32)
    Select on the adafruit microphone can remain disconnected
- Pin-number constants in the code use a `_GPIO` suffix, for example `kI2SBclk_GPIO`.
